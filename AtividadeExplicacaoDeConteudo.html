Conteúdo: Modelo TCP/IP

O Modelo TCP/IP é a base da comunicação na Internet. Ele organiza a transmissão de dados em quatro camadas, cada uma com funções específicas que permitem a comunicação entre dispositivos em redes locais e globais.

1. Camada de Acesso à Rede

A Camada de Acesso à Rede é responsável pela transmissão física dos dados através do meio físico (cabos, fibra óptica ou Wi-Fi). Ela define como os dados serão enviados e recebidos na rede local.

  • Transmissão de sinais elétricos ou ondas de rádio
  • Uso de endereço MAC
  • Controle de acesso ao meio

Exemplo: Quando dois computadores estão na mesma rede, os dados são enviados utilizando quadros Ethernet identificados por endereços físicos (MAC).

2. Camada de Internet

A Camada de Internet é responsável pelo endereçamento lógico (IP) e pelo roteamento dos pacotes entre diferentes redes.

  • Adiciona o endereço IP de origem e destino
  • Define o caminho até o destino
  • Pode fragmentar pacotes grandes

Protocolos principais: IP (IPv4 e IPv6) e ICMP.

Exemplo: Ao acessar um site, o pacote IP contém o endereço do seu computador e o endereço do servidor destino.

3. Camada de Transporte

A Camada de Transporte garante a comunicação fim a fim entre aplicações. Ela pode oferecer comunicação confiável ou rápida, dependendo do protocolo utilizado.

TCP (Transmission Control Protocol)

  • Orientado à conexão
  • Garante entrega e ordem dos dados
  • Possui controle de erro e congestionamento
  • Utiliza o processo Three-Way Handshake

UDP (User Datagram Protocol)

  • Não orientado à conexão
  • Não garante entrega
  • Mais rápido e com menos sobrecarga

Exemplo: Sites utilizam TCP para garantir que a página carregue corretamente. Já jogos online utilizam UDP para maior velocidade.

4. Camada de Aplicação

A Camada de Aplicação é onde os programas utilizam os serviços da rede. É a camada mais próxima do usuário.

  • HTTP / HTTPS - Navegação Web
  • FTP - Transferência de arquivos
  • SMTP - Envio de e-mails
  • DNS - Resolução de nomes

Exemplo: Quando você digita um endereço no navegador, o protocolo HTTP solicita a página ao servidor.
